**目标** sympy做不到的能力：猜变换→验证  
**时间** 3天 | **算力** CPU | **成本** 0

```python
class HypothesisEngine:
    strategies = [
        DetectDerivativeMatch,   # f和f'同时出现 → u=f
        DetectLinearComposite,   # f(ax+b) → u=ax+b
        DetectTrigCandidate,     # √(1±x²) → x=sinθ
        DetectReciprocal,        # 1/(xⁿ√...) → t=1/x
        DetectIBPCandidate,      # 乘积形式 → 试分部积分
        DetectLogPattern,        # f(ln x)/x → t=ln x
        DetectCompleteDerivative, # 分子是分母的导数
    ]
    def solve(expr, max_guesses=100):
        for guess in generate(expr):
            result = execute(guess)
            if verify(result, expr):  # 求导确认
                return result
        return None
```

```
1. 人类棋谱预训练           → 学会"好的棋感"

4. 打败人类的方法            → 不是模仿人类，是超越了人类
```

你的 MathAlphaZero 现在卡在哪：
```

差距不在架构，在"知识是怎么来的":
```

## AlphaGo化你的项目：真正自我学习的积分AI

### 核心思想转换

```
现在:  人写规则 → 网络学怎么选 → MCTS搜索 → 输出
以后:  积分运算的自然规则 → 网络自己发现变换 → MCTS搜索 → 输出
```

AlphaGo 不需要人手写"征子"、"倒扑"等套路——这些从自对弈中自动涌现。同样的，你的模型不应该需要人手写"分部积分"、"换元法"——应该从自对弈中自动涌现。


### 需要的6个核心改造

```
Phase A: 闭环自对弈 (阶段13-14)
  不再用"已知答案的题"训练
  而是: MCTS随机探索 → 成功/失败 → 生成经验 → 训练网络
  经验池不需要"正确答案", 只要"走到终点=+1, 卡住=0"
  
Phase B: 代数变换作为合法动作 (阶段15-16)
  不依赖"积分规则", 把变换定义为"任何代数恒等变换":
  - x → u=anything (合法换元)
  - ∫(A+B) → ∫A + ∫B (拆分)
  - f·g → 试分部积分
  - 表达式等价替换 (三角恒等式、有理分解...)
  网络自己学会哪些变换有用
  
Phase C: 端到端价值引导 (阶段17-18)
  当前网络: 判断"这个状态下胜率多少" (价值)
  升级: 判断"这个表达式离终点多近" (启发式距离)
  AlphaGo: 赢的路径越短越好
  你: 积分步骤越少越好
  
Phase D: 涌现式发现 (阶段19-21)
  不再手写规则。
  网络在自对弈中自己发现:
  "咦? 每次遇到 sin²x, 我先做 u=sin x 再...成功率很高"
  → 这个"套路"自动形成神经通路
  → 就像 AlphaGo 发现"二间高夹"一样
  
Phase E: 多机集群训练 (阶段22-24)
  AlphaGo分布式: 生成对局(CPU) + 训练网络(GPU) 并行
  你: 
  - 10台CPU机器不停自对弈 → 产生经验池
  - 1台GPU机器不停训练网络
  - 网络更新 → 回传给自对弈机器 → 生成更高质量的经验
  → 跑一周 = 现在的100万倍训练量
  
Phase F: 模型蒸馏 → 轻量推理 (阶段25-28)
  把巨量训练学到的"积分感"蒸馏到一个小模型
  本地CPU推理 → 但能力等于大模型
  像AlphaGo Lee版(单机) 已经能赢李世石
```
